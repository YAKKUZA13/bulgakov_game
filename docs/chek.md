Что ты должен увидеть в идеале:
[x] Камера видит реальный мир, объект(ы) в сцене остаются «на месте» при движении телефона.
[x] Есть обнаруженная плоскость (SLAM-плоскость при наличии, иначе depth-плоскость).
[x] Работают 3 механики: Runner, Angry, Treasure.
[x] В UI доступен Scale + Calibrate.
[x] Чекбокс SLAM points рисует точки трекинга на overlay.
Чеклист проверки (быстрый):
[+] Старт: страница открылась с HTTPS, камера разрешена, видео поток есть.
[x] Трекинг: при движении телефона объекты не «плывут» и остаются в мире.
[x] SLAM points: при включении появляются точки на overlay.
[x] Плоскость: виден debug‑меш/поверхность, физика реагирует.
[x] Runner: кубик бежит/прыгает, сталкивается с плоскостью.
[x] Angry: можно запускать снаряды, они сталкиваются с примитивами.
[x] Treasure: предметы появляются и корректно реагируют на взгляд/позицию.
[x] Масштаб: Scale и Calibrate меняют ощущаемый размер мира.
[x] Fallback: если AlvaAR не подключён, запускается fallback на сенсоры.
